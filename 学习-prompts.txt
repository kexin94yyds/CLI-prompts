mcp
一个 MCP Server 本质上是用 @modelcontextprotocol/sdk 搭建的服务端，它通过 StdioServerTransport 用标准输入输出和宿主（比如 Claude）通信，并实现两个核心请求：list_tools 用来告诉宿主有哪些工具和输入参数，call_tool 用来接收参数、执行业务逻辑并返回结果。举个例子，现有的 download_youtube_url 工具会接收一个 YouTube 链接，在系统临时目录下调用 yt-dlp 下载字幕、清洗后再返回纯文本。宿主启动进程后会先发现有哪些工具，然后在你聊天时触发调用，服务端一次性跑完整个流程、返回结果。基于这个 MCP Server，你可以扩展更多能力，比如：下载多语言字幕、搜索视频、获取视频信息、支持播放列表、截取片段、做双语对照、加时间戳方便检索，甚至组合成工作流（搜索→下载→分块→摘要），同时还能做缓存、限流和错误重试。更进一步，你也能用 MCP 框架开发完全不同的工具：比如本地文件/知识库检索、网页正文抽取、Git/GitHub 信息、Jira/Notion 事务查询、邮件日历摘要、音视频处理、DevOps 查询、系统监控等等。要从零开始，其实只需要 bun init 然后引入 SDK，写一个最小骨架（如一个 echo 工具），配置宿主用 bun src/index.ts 启动就能跑通，之后再逐步加工具、写测试、做发布。

- 连接并验证（输入密码：Linux@123）
    ssh -o PreferredAuthentications=password -o PubkeyAuthentication=no
  linux_7230264145@43.139.230.196
    登录后执行：
    clear; echo "连接验证"; whoami; sudo whoami

  常见报错一键处理

  - 被公钥拒绝（Permission denied (publickey)）
    已用强制密码参数，仍失败则重试上面命令并确认密码大小写正确。
  - Host key 报错（Host key verification failed）
    ssh-keygen -R 43.139.230.196 && ssh -o PreferredAuthentications=password -o
  PubkeyAuthentication=no linux_7230264145@43.139.230.196
  - 端口不是 22（如自定义端口 2222）
    ssh -p 2222 -o PreferredAuthentications=password -o PubkeyAuthentication=no
  linux_7230264145@43.139.230.196

  一次性加速（可选，做一次就行）

  - 建立别名后以后直接 ssh mylinux
    mkdir -p ~/.ssh && chmod 700 ~/.ssh && { echo "Host mylinux"; echo
  "  HostName 43.139.230.196"; echo "  User linux_7230264145"; echo "
  PreferredAuthentications password"; echo "  PubkeyAuthentication no"; } >>
  ~/.ssh/config && chmod 600 ~/.ssh/config
    以后连接：
    ssh mylinux
